# Streka v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Streka v1 per `docs/TAD.md`: Expo mobile app, React web dashboard, and static landing page in a pnpm monorepo, local-first with mocked services, pixel-matched to `design_handoff_streka/`.

**Architecture:** Shared TypeScript domain (`packages/core`: log-event model, streak and board derivations, Zustand stores, mock services) and design tokens (`packages/tokens`) consumed by three apps. UI code is translated screen-by-screen from the prototype HTML, which stays in-repo as the reference.

**Tech Stack:** pnpm workspaces, TypeScript strict, Zustand 5 (+persist), Vitest, Expo SDK latest + expo-router, Vite + React 19, plain HTML/CSS for landing.

## Global Constraints

- All commits on branch `staging`. No AI attribution lines in commits.
- UI copy is transcribed verbatim from the prototypes, character-for-character (including their dashes and middle dots). My own prose, comments, and commit messages never use em dashes.
- Every color/radius/size that exists as a token comes from `@streka/tokens`; no magic hex values in app code.
- Font: Archivo only. Numbers on timers/counters get tabular numerals.
- Hit targets >= 44px on mobile.
- Product rules 1-6 in `design_handoff_streka/README.md` are law; the extracted prototype logic (state shapes and formulas cited per task) defines behavior.
- Reference line numbers cited as `Proto:N` = `design_handoff_streka/Streka Prototype.dc.html` line N; `Web:N` and `Landing:N` likewise.
